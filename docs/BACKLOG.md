## Core / Infrastructure

- `[P2]` **Settings / Capture の毎フレーム再計算の最適化** — `refreshBgDropDown` /
  `refreshTargetItems()` が `onDraw` 内で毎フレーム走る。ダーティフラグでイベント駆動に。
- `[P3]` **VFS のフォルダパス形式の検討** — 区切り文字・ルート表現の方針決め (`/` 現状)。
- `[P3]` **英小文字の取り扱い方針** — 大文字中心設計。不要なら廃止し自動大文字変換等に。

## Window Manager

- `[P3]` **ウィンドウ全画面表示 (Maximize)** — ショートカット/ボタンで。既存スナップ最大化との整理。
- `[P2]` **デスクトップアイコンの右クリックメニュー** — 現状アイコン上でもランチャが出る。
  WM の `openContextMenu` を desktop.js から呼ぶ。
- `[P3]` **アイコン配置の永続化** — 並び替えた gridCol/gridRow をリロード後も保持。
- `[P3]` **アイコンのキーボードナビゲーション** — 矢印移動 / Enter 起動 / Delete。

## UI / Widgets

- `[P2]` **Collapse (折りたたみ) ウィジェット** — セクションの展開/折りたたみで領域節約。
- `[P2]` **ウィジェットのアフォーダンス改善** — Button/TextBox/NumberBox が角丸矩形で区別しづらい。
- `[P3]` **レイアウトシステムの整理** — 手動配置をより宣言的に (Grid/Flexbox 的な概念)。
- `[P3]` **BayerPicker の 2×2 対応** — 粗いがローファイ感の選択肢。
- `[P3]` **非入力要素のテキスト選択・コピー** — Label 等をドラッグ選択 → クリップボード。

## Applications

### Paint
- `[P1]` **キャンバス解像度の可変化** — 現状 128×96 固定。W/H 指定で新規バッファ生成。
- `[P1]` **1-bit 制約内での高機能化** — 選択/変形/レイヤー/フラッドフィル/ディザ塗り等。
  目指す体験: Capture → Paint 編集 → 共有 を PIXERA OS 内で完結。
- `[P2]` **ツール類のアイコン化** — テキストラベル (PEN/ERS...) をアイコンに。

### Synesta
- `[P2]` **Sampler デバイスの追加** — トラックごとに Synth / Sampler を選択可能に。
- `[P2]` **Public Domain 楽曲サンプルの同梱** — 開いてすぐ再生・編集できるデモを兼ねる。
- `[P3]` **Tab 譜機能** — 指板プレビュー付き。需要はニッチ。
- `[P3]` **簡易音声合成 (スピーチシンセ)** — 黎明期のスピーチシンセへのオマージュ。世界観要判断。

### Tessera（音の拡張 — AV ライブコーディングへ）
> 音＝時間の場 `a(t)`。チップチューン割り切り（離散的・デジタル）。生成音・`voice:` 多声・
> AV 同期・書き出し（WAV / MP4+audio）・ライブ耐性（直前 good 継続）・PERFORM（フルスク
> リーン＝画面がキャンバス＋カーソル付きオーバーレイエディタでライブコーディング）まで
> 実装済み＝音の拡張ロードマップは一巡。詳細は `lang/README.md`。
> テンポは loop 相対の `beat(n)`/`step(n)`（`bpm` は入れない）。
- `[P3]` **LangError メッセージの英語化検討** — PERFORM のエラーバー/footer は 5x5 フォントで
  日本語グリフが無く、日本語部分が欠落する。UI 言語 (英語) との一貫も含め検討。
- `[P3]` **MP4 音声の AAC プライミング補正** — AAC 先頭の encoder delay (~2112 サンプル
  ≈48ms) を edit list (elst) で補正すると AV 同期がさらに厳密になる。現状は無補正
  (実用上ほぼ知覚できない)。
- `[P3]` **外部音のリアクティブ（将来）** — SYNESTA 再生/マイクの FFT を `oscillo.js` 基盤で視覚へ。

### Life Game
- `[P2]` **有名パターンのプリセット配置** — 静物/振動子/移動体/銃を選んでクリック配置。プレビュー付き。

### Games
- `[P3]` **スコアの永続化 (VFS)** — 歴代ハイスコアを保存・表示してリプレイ動機を強化。

### About
- `[P3]` **バージョン情報ダイアログの世界観強化** — ディザ背景・ロゴ演出・隠しインタラクション。

### 新規アプリ
- `[P3]` **Console (ログ出力ウィンドウ)** — 読み取り専用 + 自動スクロール。将来の LiveCoding 出力先。
- `[P3]` **メトロノーム単体アプリ** — BPM タップ・拍子・ビートインジケータ等。
- `[P2]` **スクリーンセーバー機能** — 待機時に自動起動、任意操作で復帰。雨/マトリックス/パイプ/
  DVD バウンス/歴代名機オマージュ等。「映える画面」原則と合致。

## Visual / Design

- `[P1]` **各アプリのアイコン整備** — app-icons の未整備アプリを埋める (1-bit ピクセルアート)。
- `[P2]` **画面の明るさ調整** — 配色 RGB に輝度係数を乗算して VRAM 描画に適用。
- `[P2]` **アプリ起動時のローディング演出** — 即時表示はレトロ OS らしくない。スプラッシュ/砂時計等。
- `[P2]` **ディザグラデーション (`bayerGradRect`) の UI 装飾活用** — ヘッダ/ダイアログ背景/ホバー。
- `[P2]` **カーソル種類の充実** — ポインター/押下/I-beam/砂時計等の状態別切替。
- `[P3]` **文字装飾 (太字・下線)** — `drawText` に装飾フラグ、または Bold シート追加。

## Architecture

- `[P2]` **テストコードの拡充** — カバレッジの低いモジュール、Canvas 描画スナップショット比較等。
- `[P3]` **命名規則の "OS 的文脈" への見直し** — Web 系由来の命名を仮想 OS 語彙へ段階的に統一。
- `[P3]` **開発用デモアプリの整理** — `gradient_demo` / `easing_demo` / `ascii_art_demo` の去就判断。
- `[P3]` **TypeScript 移行** — ビルドレス原則との両立方法を検討 (JSDoc 型注釈等)。

- `[P2]` **壁紙デフォルトの変更 (SOLID → IMAGE + 同梱壁紙)** — 初見ユーザーへの引力強化。

## Ideas — 将来構想

> PRODUCT_BRIEF 記載の将来構想、およびまだ形になっていないアイデア。

- **LiveCoding 環境** — Notepad ベースにコード実行 + 音声/映像生成。Console と連携。
- **アニメーション制作ツール** — Paint 拡張。オニオンスキン/タイムライン/GIF エクスポート。
- **ゲーム制作ツール** — PICO-8 / TIC-80 的なスプライト/マップ/コードエディタ統合。
- **2-bit / モノクロ + アクセントカラー拡張** — コンセプトの根幹に関わるため慎重に。
- **キーボードショートカット一覧 / カスタマイズ**。
- **PWA 対応 (オフライン起動)**。
- **AI 機能との統合** — メロディ提案・コード補完・1-bit 画像生成等。外部 API との通信で
  ゼロ依存原則とのトレードオフあり。
- **文字列 → ASCII アートロゴ変換アプリ** — figlet/toilet 的機能。独立ウィンドウで。
- **PIXERA OS 専用レトロ Web ブラウザ + Web 圏** — 制限付き HTML 仕様 + ホスティングで
  「レトロ Web 文化圏」を形成する構想。
