# Tessera — generative-art の小さな言語

アート / CG / 数学のクリエイターのための、小さなクリエイティブ・コーディング言語。
**Tessera** = モザイクの一片（タイル）。1-bit のセルを敷き詰めて絵にする本質を表す。
拡張子 `.tess`、通称テス。tessellation（平面充填）と同語源。

**1-bit・低解像度・ディザが前提**の言語（tixy 寄り）。Processing/oF のような汎用ではなく、
レトロ制約下で最良のヴィジュアルプログラミングをするための言語。**1-bit はホスト差し替え
可能なディテールではなく言語の前提**（媒体非依存＝色/高解像度 は目標ではない）。

**独立性＝デプロイ独立**（媒体非依存ではない）。言語コアは SYNESTA アプリに hard 依存しない
ので、SYNESTA 内の TESSERA アプリでも、スタンドアロンの公開 Web エディタでも、**同じ
SYNESTA 1-bit canvas スタック**で同等の体験を提供できる（OSS 公開もあり得る）。

## 北極星
- **low floor, high ceiling** — tixy / p5.js のような入りやすさ＋上級者の高い天井。
- **慣習・正確さ優先の命名** — GLSL / Processing / 数学の伝統に従い、造語を避ける
  （`mix` `clamp` `fract` `smoothstep` `dist` `noise` `fbm`
  `worley`（セルラー＝voronoi/stone 風）…）。
  世界観・詩情は判断材料にしない。対象読者にとって最小の驚きを最優先。
- **可読性 > 効率** — 1-bit・40桁の小さな作品は「読んで分かる」ことが価値。多少の再計算より
  明快さを採る。長い項・引数は意味ある名前で中間変数化し（`d`=距離 / `g`=グレーティング /
  `cx,cy`=中心 / `u,v`=ワープ座標 / `w`=波 / `p,q`=位相・角 / `s`=累算）、1 行 1 概念へ分解する。
  数値は `.5` 形（先頭/末尾ゼロを落とす＝GLSL/tixy 慣習）。これらは整形(`format.js`)とサンプルが
  体現する: 機械的な部分（`.5`・整列・40桁折返し）は整形が、命名は作者が担う。 — stdlib は素の値を返す（`sin` は [-1,1]）。0..1 正規化や Bayer ディザ等の
  ラスタライズは**別レイヤ（共有レンダラ `core/field_render.js`）**が担う（内部規律。媒体
  非依存の主張ではない）。表示方式は `view:` でコードから宣言でき、レンダラが解釈する。
- **決定論** — `seed` で完全再現。コードそのものが作品＝レシピ（表示方式も `view:` で含められる）。
- **大小文字を区別しない** — SYNESTA は表示が常に大文字なので `CANVAS` と `canvas` が同じ見た目に
  なる。混乱を避けるため識別子は大小無視で一致させる（lexer が小文字へ畳む。整形は字面を保つ）。
  ホスト側のエディタも入力を大文字化する（入力＝表示＝パースを一致）。

## 言語の形 — 場 f(x,y,t)
プログラムは場 `f(x,y,t)` を表す**式 1 本**（`f(x,y,t) =` ヘッダは任意）。x,y ∈ [0,1] を
全セルで評価し、返り値（インク level 0..1）を 1-bit へディザする。状態を持たない純関数なので、
任意の (x,y,t) を直接サンプルでき、解像度非依存（プレビュー / 任意サイズ書き出しが軽い）。

- **値ブロック**: 最終式の前に `代入` / `repeat { … }` 文を置ける（セル毎に評価）。
  セル内反復・総和で **julia / quasicrystal / metaball** 等の閉形式を書ける
  （例: `m=0` ＋ `repeat 24 { … m=m+… }` ＋ 最終式 `clamp(1-m/200,0,1)`）。
  発散しうる反復は `clamp` で抑える（escape/break は未実装＝予算固定）。

**設定ディレクティブ** — トップレベルの軽量宣言（順不同・任意・省略時は既定）で、表示方式・
出力サイズ・乱数まで**コードに含められる**（recipe 自己完結・p5 的）。`repeat` の `{…}` 内
（ブレース内）は走査しないので衝突しない。

- `view:` （例 `view: halftone(8)`）— 表示方式と数値パラメータ
  （dither/hatch/halftone/braille/ascii ＝ 1-bit で映える「面」系）
- `canvas:` （例 `canvas: 1920x1080`）— 出力解像度（外寸px。`1920 1080` でも可）。
  base(ドット数)=canvas/8。プレビューもこのアスペクト比
- `pad:` （例 `pad: 64`）— 額縁マット（出力px）
- `fps:` （例 `fps: 20`）— アニメ/書き出しのフレームレート
  （ホストは 100 の約数へスナップ）
- `period:` （例 `period: tau`）— ループ周期秒
  （定数式可。プレビュー周回・GIF/MP4 のシームレスループ長）
- `seed:` （例 `seed: 42`）— 乱数シード（rnd/noise/fbm を振り直す）

> **pixel は無し**。1 アートドット = **8 出力px 固定**（チャンキー 1bit が核）。base(ドット数)=canvas/8。

コアはこれらを**不透明データ**として持つだけで**ラスタライズ・適用しない**。既定値・範囲クランプ・
実際の出力はホスト（TESSERA / `art_export`）の責務。未指定は各ホストの既定（TESSERA は
`view:dither(2)` / `canvas:1080x1080` / `pad:80` / `fps:20` / `seed:0` / `period:tau`）。

## 音 — 時間の場 a(t)（任意・チップチューン割り切り）
視覚が「空間の場 `f(x,y,t)`」なら、音は「**時間の場 `a(t) → [-1,1]`**」。トップレベルの
`sound:` 以降が音の場で、視覚と**同じ式言語**（値ブロック・`repeat` も同じ）で書ける＝
別パラダイムを増やさない。`sound:` が無ければ無音（視覚は不変）。決定論的で `period` で
きっちりループするので、ホストは 1 周期をオフラインレンダしてループ再生／書き出せる。

**サウンドは 1-bit と同様「離散的・デジタル（チップチューン）」に割り切る**（連続フィルタ等の
アナログ的音作りは持たない）。語彙（純関数。オシレータ/拍は音クロック `_at`/`period` を暗黙に読む）:

- `hz(semi)` — 半音番号 → 周波数（A4=69=440Hz）
- `pulse(freq, duty=.5)` / `tri(freq)` / `saw(freq)` — 矩形/三角/鋸（出力 [-1,1]）
- `nz(rate)` — サンプル&ホールド白色ノイズ（`rate` Hz・seed 連動）。※視覚の `noise`(2D 値ノイズ) とは別
- `beat(n)` — 1 周期に n 回リセットする鋸位相 0..1 / `step(n)` — 1 周期を n 分割した整数 0..n-1
- `seq(i, ...xs)` — `xs[i mod len]`（音程・値の並び。可変長＝配列リテラル不要）
- `decay(p)` — 拍位相 p を 1→0 の減衰包絡へ

**音色ディレクティブ** `voice <名前>: <式>` — 名前付き音色（＝トラック名）を宣言する。式は
`f`（周波数）を参照でき、音の場から `名前(freq)` で呼ぶと `f` が束縛される。**上で音作り・下で
作曲**に分離でき、多声は `+` で混ぜる（別スケジューラは持たない＝場のモデルのまま）。

例（3 声のチップチューンループ。矩形波の旋律＋三角波ベース＋ノイズのハット）:

```
voice lead: pulse(f, .25)
voice bass: tri(f)
voice hat:  nz(1000)

sin(x*8 - t)*.5 + .5                    // 視覚: 空間の場

sound:                                  // 音: 時間の場（値ブロックで多声）
  m = lead(hz(60 + seq(step(8), 0, 3, 7, 10))) * decay(beat(8))
  b = bass(hz(36))
  h = hat() * decay(beat(16))
  m*.4 + b*.4 + h*.2
```

> **テンポ観**: 拍は `beat(n)`/`step(n)` ＝ **loop 相対（1 周期に n 回）** で tempo 非依存。
> 「すべてが period でループする」TESSERA の identity・AV ループと自然に一致するため、
> 絶対時間の `bpm` は入れない（二重クロックの混乱を避ける）。

> **現状 P2**: 生成音（`sound:`）のループ再生＋名前付き音色（`voice:`）・多声まで。TESSERA では
> `Alt+P` で再生/停止（`period` 同期）。今後: 視覚が音を読む AV 同期（`amp` 等）、
> 音付き書き出し（WAV / MP4+audio）。

## アーキテクチャ（薄いコア → 複数デプロイ）
言語ランタイムは抽象「サーフェス」契約（`surface.js`）に対してのみ描画する。デプロイ先
（SYNESTA アプリ / 公開 Web エディタ）が契約を実装する。両デプロイとも**同じ 1-bit canvas
スタック**を使う想定（媒体差し替えではなく配布形態の差）。

- 言語本体（core/stdlib/runtime）はデプロイ先を import しない＝描画なしで単体テスト可能・
  スタンドアロン配布可能。これが「独立」の実体。
- ラスタライズ（0..1 → 1-bit、dither/hatch/…）は共有レンダラ `js/core/field_render.js` が担う
  （TESSERA が使う。将来の別デプロイでも共有できる設計）。
- `playground/`（canvas2D + HTML の薄いシム）は当面の**開発ハーネス**。公開 Web エディタの
  本命は「SYNESTA スタックをそのまま canvas 内 UI で出すスタンドアロン版」（将来）。

```
/lang
  README.md          ← これ
  surface.js         ← 抽象サーフェス契約 + ディザ + makeBufferSurface（1-bit FB）
  core/  lexer parser interp     ← 言語コア（依存ゼロ）
  stdlib.js          ← 関数ボキャブラリ（純関数。surface 非依存）
  runtime.js         ← ソース → 場へコンパイル
  format.js          ← 整形（TESS 専用。40桁折返し・ディレクティブ/代入整列・コメント正規化）
  smoke.mjs          ← node 動作確認
  playground/        ← ブラウザ開発ハーネス（エディタ + ライブプレビュー）
```

SYNESTA 統合: `js/app/tessera.js`（TESSERA）が唯一の creative-coding ウィンドウ。場は
**共有レンダラ**で 1-bit 化：ピクセル方式は `js/core/field_render.js`、ASCII は
`js/core/ascii_art.js` の `renderAsciiLines`（場→文字グリッド）。表示方式・出力は**すべてコードの
設定ディレクティブ**で指定し、画面のコントロールは「形式 + DL」のみ＝最小。プレビューは `canvas:` の
アスペクト比を反映し `.buf` を GPU.blit。書き出し（合成→PNG/GIF/MP4）は
共有 `js/core/art_export.js`。VFS 連携（`.tess` 保存/読込、サンプルは `/Sketches/Learn`（番号順チュートリアル）と `/Sketches/Gallery`（作例））。
Ctrl+E 書き出し / Ctrl+R シード振り直し / Shift+Alt+F 整形。エディタは 40桁（点線ガイド表示、
超過行は右端に実線ティック）。Shift+Alt+F は TESS 専用フォーマッタ（`format.js`）で、40桁を
制約として尊重する: 長い式を `()` 内の低優先 `+ -`（無ければ `* /`）で折り、深度0の式は
グルーピング括弧を補ってから折る（値不変）。ディレクティブは正準順（canvas→pad→fps→
period→seed→view）へ整列、連続代入の `=` も揃える。折ってもなお 40桁に入らない密な式
（comma だけの呼び出し等）はそのまま 1 行＝エディタのティックが知らせる。

## 現状
**「場」の単一パラダイム**：視覚＝空間の場 `f(x,y,t)`、音＝時間の場 `a(t)`（同じ式言語）。
SYNESTA との **場レンダラ共有（北極星 B）** はピクセル方式（`core/field_render.js`）＋
ASCII（`core/ascii_art.js` の `renderAsciiLines`）まで完了。音は P2（生成音のループ再生＋
`voice:` 名前付き音色・多声）まで。

**シンプル化（言語を美しく保つ）**：1-bit アート的に映えない・教材化しにくい機能を削除。
- **draw（手続き的な point/line）と cells（状態を持つ反応拡散/CA）を廃止し、場一本に統合**＝
  実行モデル 1 つ・任意 (x,y,t) サンプル可・warmup 不要。好みの作品はすべて場で書ける、という
  判断（数学的に高度でも 1-bit アートとして弱い算法はすべて draw / cells 側だった）。
- 線ベースの表示方式 `contour` / `scanline` ＝ 1-bit で映えない、を廃止（面系のみ残す）。
